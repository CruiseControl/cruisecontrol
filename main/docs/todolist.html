Help Wanted

The CruiseControl Project can use your help. Following are suggestions for features. Submissions and additional feature requests should be directed to the cruisecontrol-devel@lists.sourceforge.net mailing list.

To Do List:

  • Pull config.xml changes from version control before a build
  • Store dynamic changes into version controlled config.xml

SourceControl Unit Tests
CruiseControl supports interactions with multiple source control tools. Unfortunately, our unit tests are a bit under developed. Most of our source control wrappers just parse the output from the command line, so capturing that output to a text file gives us input that we can use for our unit tests. Building our unit tests around this will help us to manage changes between versions, locales, etc.

TextEmailPublisher
Currently, CruiseControl has the feature of sending an email notification. Currently this email message is limited to a link to the build results JSP. Having a brief text summary in the body of the email has been suggested. See publishers.EmailPublisher and publishers.LinkEmailPublisher.

Documentation
A project is only as good as it's documentation. If any inconsistencies/inaccuracies are found, please report them to us. Any other suggestions for improvement are encouraged as well.
