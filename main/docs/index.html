The CruiseControl Build Loop

What Is It?
The Build Loop is designed to run as a daemon process which will periodically check your source control tool for changes to your codebase, build if necessary, and send out a notification regarding the status of the build.

How Does It Work?
CruiseControl defines a build cycle - determine if a build is necessary, build, create a log file, and send notifications. The daemon process will wake up at a user defined time interval and attempt a build cycle. Each time, the configuration file will be reloaded so that the CruiseControl process need not be restarted for the changes to take effect. At the end of the build, a log file will be written for use by the Build Results JSP.

 

New Features

XML Configuration
Defining the configuration for CruiseControl in XML has allowed us freedoms that were not easily possible with the old properties file.

Runs standard Ant build files
CruiseControl now runs as a true wrapper to Ant and no longer requires any changes to your existing build file to run.

Pluggable Architecture
CruiseControl is easier to customize now that almost all of the non-core functionality is implemented as plugins. See the plugins page for more details about the plugin philosophy and the config.xml page for more details about the stock plugins that ship with CruiseControl.

Better Scheduling Functionality
Schedule as many different builds as you like, run at an interval or scheduled at a given time.

HTML email support
View the build results directly on the notification emails rather than linking to the build results JSP.

Upgrade utility for migrating from 1.x to 2.0
Relatively painless upgrading if you have an earlier version of CruiseControl installed. Run the included Upgrader and be ready to run 2.0 almost instantaneously.

Better Memory Management
The CruiseControl 1.x product line was plagued by memory leaks. Most should be plugged now, and the decoupling from Ant should make sure that CruiseControl's memory usage should not grow over time.
