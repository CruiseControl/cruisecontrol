Installing the main build loop.
  • 1. Download src/build
  • 2. Create config.xml file
  • 3. Startup cruisecontrol
1. Download Source/Build

Download the source from CVS. If you have CVS installed, the command to download is:

cvs -z3 -d:pserver:anonymous@cvs.cruisecontrol.sourceforge.net:/cvsroot/cruisecontrol co cruisecontrol

When prompted for the password for anonymous, simply press the <ENTER> key. If you examine the code you just downloaded, there will be a build.bat (for Windows users) and a build.sh (for UNIX users) present in the cruisecontrol directory. Execute the appropriate script, and CruiseControl should compile. Look in the dist directory, and you should find a cruisecontrol.jar file if the build was successful.

 2. Create config.xml file / Upgrade previous version

All of the cruisecontrol configuration exists in the config.xml file. If you're new to cruisecontrol, you'll have to create this file. If you have used a previous version of cruisecontrol, you can use the upgrader to convert your old setup to a config.xml file.

New users should read the explanation of the config.xml file format

To run the upgrader, do the following:

  • Copy dist/cruisecontrol.jar to the top-level CruiseControl directory. This is necessary because cruisecontrol.jar assumes this location for its classpath.
  • Execute the following command:

java -jar cruisecontrol.jar -upgrade buildfile propertiesfile configfile projectname

Where the command line attributes are as follows:
Flag Required Description
buildfile Yes build.xml file that contains the modificationset task. The buildfile will not be modified.
propertiesfile Yes cruisecontrol.properties file from the previous installation.
configfile Yes New config file to be written. Suggested naming convention of config.xml
projectname Yes Unique idenifier for this project.

A config file should be written to disk, and we can proceed to the next installation step.

Note: An alternative approach that would not require moving the cruisecontrol.jar would be to modify the CCDIR variable in the cruisecontrol.bat or cruisecontrol.sh scripts, also located in the dist directory.

3. Startup CruiseControl

CruiseControl can be started using two approaches:

  • Scripts. cruisecontrol.bat or cruisecontrol.sh. Modify value of CCDIR as appropriate
  • Executable jar. Copy jar to <cruisecontrol> directory first and type:

    • java -jar cruisecontrol.jar

In either approach, the following command line arguments can be specified:
Flag Required Description
-projectname name Yes Should match the name attribute of the <project> element in config.xml
-lastbuild timestamp First Time Timestamp format is YYYYMMddHHmmss
-label label First Time Label format depends on the labelincrementer used.
-config filename First Time Relative path to config.xml file

After the first build cycle, the values for-lastbuild, -label and -config will be saved. If cruisecontrol should need to be restarted, the values for these arguments need not be specified, unless you would like to override the values that cruisecontrol has. The only argument that is always required is -projectname.
