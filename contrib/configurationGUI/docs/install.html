CruiseControl Configuration Client Installation

Version 0.9

This client may be used to edit the cruisecontrol.xml file or create new one.

Installs:

  • As a webstart application from the CruiseControl site.
  • As a webstart application from a local web container.
  • As a standalone application.

From CruiseControl Site

Use this method if you want to keep up to date with the latest changes and you have internet access.

To Install:

  1. Click here.
  2. Select Ok to open with default application.
  3. Java Webstart will then download several files.
  4. You will be asked if you want to trust the application. Click Start button.

Local Webstart install

The local webstart install is for a large team that wants to keep the install on the corporate network. This install requires a web container like Tomcat to deploy the install inside of.

To Install:

  1. Run ant build.webstart.dist from the root directory. This will build the cc-configuration.war file in the dist sub-directory..
  2. Deploy the cc-configuration.war file in your web container. For Tomcat users, just drop the .war file in the webapps directory.
  3. Point your web browser to http://yourserver/cc-configuration/cruisecontrol-gui.jnlp.
  4. Select Ok to open with default application.
  5. Java Webstart will then download several files.
  6. You will be asked if you want to trust the application. Click Start button.

Standalone Client Application

This install is for those who may not have internet access or only want to run the current version and don't want automatic updates.

To Install:

  1. Run ant build.standalone.dist to create the standalone.zip file in the dist sub-directory.
  2. Unzip the standalone.zip file into whichever directory you wish to run it from.
  3. Use the startup.bat or startup.sh file in the bin sub-directory to run the application.
